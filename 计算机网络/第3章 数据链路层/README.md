数据链路层
==========
##进一步分层
IEEE将数据链路层进一步划分：
* 逻辑链路控制（Logical Link Control），即LLC。
* 媒体接入控制（Media Access Control）,即MAC。比如：
  * 以太网MAC
  * 令牌环MAC
  * 令牌总线MAC
  
MAC层与物理层相邻，不同的物理层介质对应着不同类型的MAC。  
LLC层在MAC层之上与网络层相邻，不同的MAC层都对应着统一的LLC。
##帧
数据链路层的通信单位是帧
##MAC地址
又称数据链路地址、物理地址。不同的物理层，它们的MAC地址的类型是不同的。
###以太网MAC
为6字节（48位）长度，通常写成：冒号分十六进制的格式。比如：｀１C:75:08:3B:5E:EC｀
###单播、多播、广播
一个MAC帧包含目的地址和源地址。源地址一定是单播的，目的地址可能是单播、多播甚至广播。  

||区别|
|---|---|
|单播|目的地址的第一个字节的最低位是0。即偶数
|多播|目的地址的第一个自己的最低位是1。即奇数
|广播|目的地址的所有位都是1，即`FF:FF:FF:FF:FF:FF`
第一个字节指的是右数第一个字节。数字的位数都是从右往左数的，很好理解。
