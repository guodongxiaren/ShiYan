关系数据库
===========
##重要术语
###候选码
若关系中的某一属性组能唯一的标识一个元组，则称该属性组为候选码（Candidate key）。
###主码
若一个关系有多个候选码，则选定其中一个为主码（Primary key）。
###主属性
候选码的诸属性称为主属性（Prime attribute）。
###非主属性
不包含在任何候选码中的属性称为非主属性（Non-prime attribute）。
###全码
极端情况下，关系模式的所有属性是这个关系模式的候选码，称为全码（All-key）。

##关系基本操作
- 选择
- 投影
- 并
- 差
- 笛卡尔积

其他的**连接、除、交**都可以用这5个基本操作推导出来。
